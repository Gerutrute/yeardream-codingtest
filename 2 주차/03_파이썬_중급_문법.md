# 1. 함수와 메서드


## 함수
특정 기능을 수행하는 코드들의 모임

```
라면끓이기() 함수 실행

> 냄비를 가스레인지에 올리기, 가스레인지 불 켜기, 냄비에 물을 붓기 ...

```


## 함수의 종류
### 내장 함수
- 파이썬 개발자들이 이미 만들어 둔 함수들이며 편리하게 가져다 사용하면 됩니다.

#### input()
자료를 입력하는 함수

#### print()
자료를 출력하는 함수

```python
a = input()

print(a)
```

#### max()
시퀀스 자료의 최댓값을 구하는 함수

#### min()
시퀀스 자료의 최솟값을 구하는 함수


#### sum()
숫자 원소로 이루어진 시퀀스 자료의 합


#### len()
시퀀스 자료의 길이를 구하는 함수


### 사용자 정의 함수
사용자가 여러 코드를 묶어서 새로 만든 함수

```python
def 인사(사람):
    return f"📣 {사람}님, 어서오세요! 오늘도 반가워요 :)"

print("김동근")

> 김동근님, 어서오세요! 오늘도 반가워요 :)
```


### 함수 정의
define(정의하다) 키워드를 이용해서 함수 정의합니다.

```python
def 함수이름(매개변수): 
    <수행할 명령>
    ...
    return 반환값
```


## 지역 변수와 전역변수
### 지역 변수
- 함수 안에서 일어난 일은 함수 밖에 영향을 미치지 못합니다

```python 
def function(a):
    x = "Hi!"
    print(a)

a = 3
function(a)

print(x) # function() 함수 안에서 선언된 x라는 변수는 찾을 수 없음!
```

### 전역 변수
```python
x = "Hi!" # x 변수 등장

def func(a):
    print(a)

func(x) 
print(x) # 어디서든 사용 가능한 x
```


## 메서드
### 메서드 (Method)
특정 자료에 대해 특정 기능을 하는 코드에요

```python
list = [1, 2, 3]

list.append(4)
list.count(2)
list.pop()
```


### 함수 vs 메서드
- 함수 : 주어진 특정 기능을 하고 매개변수에 자료를 전달합니다.
- 메서드 : 특정 자료와 연관 지어 기능을 하고 자료 뒤에 .을 찍어 사용합니다

```python
list = [1, 2, 3]

len(list) # len 함수
sum(list) # sum 함수


list.append(4) # append 메서드
list.count(2) # count 메서드
```



# 2. 모듈과 패키지

## 모듈
- 특정 목적을 가진 함수와 자료의 모임

### 필요성?
- 코드의 길이가 길어지는 상황
- 모든 함수, 변수를 구현하는 것은 불가능
- 누군가 만들어놓은 함수와 변수, 자료를 활용해야 함



## 모듈 사용하기
import(불러오다) 키워드를 이용해서 모듈 사용합니다.

```python
import random # random 이라는 모듈 불러오기
```


- .을 쓴 후 모듈 속 함수/변수 사용
```python
import random

print(random.randrange(0, 2)) # 0 ~ 2 까지 랜덤 선택

> 1
```

### 모듈 만들기
원하는 내용을 담아서 .py(파이썬 파일)로 만들 수 있음

```python
# cal.py
def plus(a, b):
    c = a + b
    return c


# main.py
import cal

print(cal.plus(1, 2))
```


## 모듈 활용하기
### math
수학 연산을 도와주는 모듈
```python
import math # math 모듈 불러오기
```

### random
랜덤한 숫자를 선택해 주는 함수들이 담긴 모듈

```python
import random # random 모듈 불러오기
```


## 모듈과 패키지
### 패키지 (Package)
모듈을 폴더 (Directory)로 구분하여 관리하는 것

### 왜 써?
- 모듈이 많아지면 관리가 어려움

- 기능별로 폴더로 정리하면 가독성, 유지보수가 쉬움

```
my_project/
│
├── main.py
└── calculator/            ← 패키지 (폴더)
    ├── __init__.py        ← 패키지임을 알려주는 파일
    ├── add.py             ← 모듈
    └── subtract.py        ← 모듈
```

- from - import 를 사용해서 함수/변수 사용할 때 .을 사용하지 않아도 됨

```python
from user.cal import plus

print(plus(3, 4))
```
